1. Prinsip SOLID: 
- Single Responsibility Principle (SRP) => sebuah class hanya bertanggung jawab untuk mengerjakan satu pekerjaan saja. Saya mengaplikasikannya pada 
- Open-Closed Principle (OCP) => kode terbuka untuk di extend tanpa harus mengubah kode asli. 
- Liskov Substitution Principle (LSP) => sebuah subclass dapat menggantikan peran superclassnya
- Interface Segregation Principle (ISP) => sebuah interface hanya untuk mengerjakan satu hal
- Dependancy Inversion Principle (DIP) => sebuah class bergantung pada abstract class atau interface. 

2. Kelebihan mengaplikasikan prinsip SOLID 
- kode menjadi lebih fleksibel. 
- kita lebih mudah meningkatkan maintainability dan comprehensibility
- mudah mencari bug
- lebih mudah untuk menambah fitur baru. terlebih dari prinsip open-closed principle yang membuatnya 

3. Kekurangan tidak mengaplikasikan prinsip SOLID
- kode lebih rentan mendapatkan error 
- kode lebih sulit untuk dipahami 
- proses pengecekan bug lebih sulit dilakukan
